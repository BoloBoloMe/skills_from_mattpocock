# 深模块

来自《软件设计哲学》（A Philosophy of Software Design）：

**深模块** = 小接口 + 大量实现

```
┌─────────────────────┐
│       小接口        │  ← 方法少，参数简单
├─────────────────────┤
│                     │
│                     │
│       深实现        │  ← 复杂逻辑被隐藏
│                     │
│                     │
└─────────────────────┘
```

**浅模块** = 大接口 + 少量实现（避免）

```
┌─────────────────────────────────┐
│             大接口              │  ← 方法多，参数复杂
├─────────────────────────────────┤
│             薄实现              │  ← 只是透传
└─────────────────────────────────┘
```

设计接口时，问自己：

- 我能减少方法数量吗？
- 我能简化参数吗？
- 我能把更多复杂度隐藏到内部吗？
