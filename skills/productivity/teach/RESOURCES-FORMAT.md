# RESOURCES.md 格式

`RESOURCES.md` 是此主题的可信来源精选集. 解释器的知识应从这里抽取, 不是从参数化猜测. 智慧来自此处列出的社区.

## 结构

```md
# {Topic} Resources

## Knowledge

- [Book: _The Science and Practice of Strength Training_ - Zatsiorsky & Kraemer](https://example.com)
  关于编程和适应的基础文本. 用于: 关于 periodisation, recovery, intensity zones 的任何东西.
- [Article: "How Much Should I Train?" - Greg Nuckols (Stronger By Science)](https://example.com)
  关于 volume landmarks 的循证综述. 用于: 每个肌肉群每周 set 目标.

## Wisdom (Communities)

- [r/weightroom](https://reddit.com/r/weightroom)
  高信噪比 subreddit, 经过反 bro-science 审核. 用于: program 评论, plateau 排错.
- Local: Tuesday strength class at {gym name}
  用于: 对 lift 的实时教练反馈.
```

## 规则

- **仅高信任.** 偏好原始来源, 公认专家, 同行评审工作, 以及有强审核的社区. 如果一个资源是营销伪装成教育, 排除它.
- **为每条注释.** 三个月后, 一条裸链无用. 加一行: 它涵盖什么以及何时使用它.
- **按 Knowledge / Wisdom 分组.** 映射 [SKILL.md](./SKILL.md) 中的理念. 资源仅出现在一组中也可以.
- **明确展示空白.** 如果没有好的资源为使命需要的某个领域存在, 写一个 `## Gaps` 章节列出缺失. 这驱动未来搜索.
- **无情剪枝.** 一个被证明为错, 浅薄或偏离使命的资源应被移除, 不是埋藏. 五个尖锐来源胜过三十个平庸.
- **记录社区偏好.** 如果用户选择不加入社区, 在此注明以便未来会话不再持续提议它们.
